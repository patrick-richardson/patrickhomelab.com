External Access Architecture

Overview

External access to PatrickHomeLab is designed to provide secure HTTPS connectivity to selected services without exposing internal network ports directly to the internet.

Traffic Flow

User → Cloudflare → Tunnel → proxy-01 → Caddy → Internal Services

Components

The external access system is built using a layered approach, with each component responsible for a specific part of the ingress and routing process.

Cloudflare: Handles DNS, HTTPS termination, and acts as the external entry point into the environment.

Cloudflare Tunnel: Maintains a secure outbound connection from proxy-01 to Cloudflare, eliminating the need for inbound port forwarding.

proxy-01: Dedicated ingress node that receives tunneled traffic and serves as the central entry point into the lab.

Caddy: Reverse proxy responsible for routing incoming requests to the appropriate internal services.

Internal Services: Includes monitoring, media, and other lab systems that are selectively exposed through the proxy layer.

Design Rationale

This architecture was designed to provide secure external access while minimizing exposure of the internal network. Instead of opening ports on the router, all inbound traffic is handled through an outbound tunnel initiated from within the lab environment.

By using a reverse proxy as a centralized entry point, all service routing is managed in a single location. This simplifies service exposure, improves scalability, and allows additional services to be added without modifying network-level configurations.

Separating public access from internal infrastructure ensures that only intended services are exposed, while monitoring, automation, and storage systems remain isolated within the private network.
